# Research Workflow

## Research Order
1. Define scope — Confirm topic, purpose, depth (use `deep-interview` if needed)
2. Collect data — Web search, codebase exploration (`explore`), document analysis
3. Analyze/organize — Extract key insights, comparative analysis, structure
4. Visualize — Use `visualize` for charts/diagrams/timelines when needed
5. Write report — Generate final deliverable via `pdf`/`docx`
6. Deliver — Share file via `--sendfile`

## Principles
- If scope is unclear, ask before starting
- Set one core question per research task
- Prioritize primary sources (original documents, official docs)
- Clearly distinguish speculation from facts
- Keep interim results organized and reusable
