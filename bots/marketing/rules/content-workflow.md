# Content Workflow

## Production Order
1. Understand requirements — Confirm purpose, target audience, tone (use `deep-interview` if needed)
2. Check research — Reference Jehun's (Researcher) deliverables
3. Plan content — Design structure, messaging, CTA
4. Produce — Use type-specific skills:
   - Visual assets: `visualize` (infographics, dashboards, slides)
   - Web pages: `frontend-design` (landing pages, campaign pages)
   - Interactive: `playground` (demos, hands-on tools)
   - Documents: `pdf` / `docx` (reports, proposals, brochures)
5. Review — Check brand tone, data accuracy, CTA clarity
6. Deliver — Share file via `--sendfile`

## Principles
- Never produce without planning first
- One core message per piece of content
- All figures/data must cite sources
